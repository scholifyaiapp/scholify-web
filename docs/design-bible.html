Scholify Design Bible

The single source of truth for how Scholify looks, moves and speaks — every token, component and rule here is pulled verbatim from the shipped foundation (ui.tsx, charts.tsx, index.css). Toggle your system or in-app theme — every swatch below is wired to the real --sch-* tokens, light and dark.

§0

Brand mark & wordmark

Six rotational circuit traces closing into a hexagon — each terminal a node, each turn a waypoint on the journey to graduation. It is the Loop, drawn. One arm rotated 6×60° around center. Primary variant: MONO / DEEP RED. Minimum size 16px absolute floor; clear space of one node diameter on every side.

red — primary
white — dark knockout

wordmark — Plus Jakarta Sans 700

Never: stretch or distort · recolor off-brand · add shadows or 3D · place on low contrast · render below 16px · rebuild the mark by hand.

§7

Data-viz rules

bandColor(pct, passLine = 60) — the one status formula. Never used for identity, only for a number against a real target.

ConditionColorMeaning
pct ≥ passLineGreengood
pct ≥ passLine − 15Amberat-risk
otherwiseRedbehind

Values are always visible as text — color is never the only channel. Targets are drawn, not implied. Empty states teach ("Sit two timed mocks and the trend… draws itself") — never a bare dash.

§8

Motion

House easing [0.22, 1, 0.36, 1]. Rings draw in 1.1s, meters 0.8s, list rows stagger by 0.05s. Hover lifts (y:-2) and never shifts layout; tap scales to 0.98–0.99. Maximum one ambient motion element per screen. prefers-reduced-motion zeroes everything, globally.

§9

Voice & copy

Charles is a coach with receipts: warm, direct, always tying claims to the learner's own numbers. Recovery language — never "failed."

§10

Landing-only rules

The landing is pinned — layout and section structure are frozen, changes are additive only. It runs its own token set (--bg-primary: #FAFAF7, brand/plum/fire/shield) and its own fonts — Instrument Serif (display), Geist (sans), Geist Mono, Caveat (handwritten) — and does not theme dark. EN/RU parity is mandatory for every user-visible string; ACCA paper names and codes stay English in both. Signature effects (mesh background, conic borders, glass pills, marquee, 3D tilt) are landing-only and never migrate into the app shell.

§11

Checklist for any new screen

  • Built from ui.tsx + charts.tsx — no hand-rolled cards, buttons, bars or rings.
  • Colors via C / --sch-*; status colors only against a real pass line.
  • Spacing on the SP grid; radii from R; shadows from SHADOW.
  • Icons via Icon/IconBadge semantic names; zero emoji-as-icons.
  • One hero number; values as text; targets drawn and named.
  • rise on entry, staggered lists, tap feedback, no hover layout shift, reduced-motion safe.
  • Honest empty/measuring state; fallback labels for keyless AI.
  • 44px touch targets; truncation-safe; auto-fit grids; wide content scrolls in its own container.
  • Copy: evidence-tied, sentence case, uppercase labels, recovery language — never "you failed."

AI design prompt

A condensed version of everything above, meant to be pasted in front of any AI request to generate or review a Scholify screen — the guardrails without the full bible.

You are designing/building a screen for Scholify, an ACCA exam-prep app. Follow its shipped design system exactly — do not invent new patterns.

BRAND
- Mark: hexagonal circuit of 6 rotated waypoints. Primary variant MONO / DEEP RED #C80000. Never distort, recolor, shadow, or rebuild by hand.
- Gradient: linear-gradient(135deg, #C80000 0%, #E50068 48%, #F4A405 100%) — ONE gradient CTA per screen, on buttons/active-tabs/headline text only. Never on body text or data marks.

COLOR
- Everything through tokens, never raw hex for surfaces/text: bg, card, card-2 (inset well), text, tx-1..4 (muted→ghost), border, hairline.
- Brand red = identity/interaction only, NOT status.
- Status color exists only against a real target: bandColor(pct, passLine=60) → green (pct>=passLine) / amber (pct>=passLine-15) / red (below). Their *Soft tints are the only allowed tinted chip backgrounds.

TYPE — Plus Jakarta Sans (app) / Instrument Serif+Geist (landing only, separate system)
h1 26/800 -2%   h2 20/800 -1%   h3 16/750   body 14/450   small 12.5/500   label 11.5/800 uppercase +6%
Sentence case everywhere except uppercase section labels. Never title-case body copy.

SPACING / SHAPE
4px grid: 4,8,12,16,20,24,32,40. Card padding = 20. Radii: sm10 md12 lg14 xl16 2xl20 pill999. Cards use 2xl, buttons lg, chips pill.
Shadows are warm-ink-tinted (rgba(51,43,40,..)), never pure black. sm=resting, md=hover, lg=modals, brand=the one CTA on screen.

ICONS
Lucide only, via a semantic name map (never inline-import a raw icon in a screen). Zero emoji as icons anywhere.

COMPONENTS — reuse, never hand-roll
Card, Button (primary=gradient/secondary=bordered/ghost=transparent, 44px min-height, tap-scale 0.98, no hover layout shift), Badge (pill status chip), Stat/StatCard, RingGauge (donut + count-up + target tick), MeterBar (target tick), BreakdownList, TrendBars, Sparkbars, DeltaChip.

DATA-VIZ
One hero number per view. Values always shown as text, color is a second channel only. Targets are physically drawn AND named in text ("pass line 60%"). Empty states teach what starts the measurement — never a bare dash or fake data.

MOTION
Easing [0.22,1,0.36,1]. Entrances: rise (fade+10px) for sections, staggered 0.05s for list rows. Hover lifts (y:-2) or changes color/shadow — NEVER shifts layout or scales. Tap = scale 0.98-0.99. Data draws in once on mount (rings 1.1s, meters 0.8s). Max ONE ambient/looping motion element per screen. Respect prefers-reduced-motion globally, no exceptions.

VOICE — "a coach with receipts"
Every big claim quotes the number ("You're at 43% to pass FR"). Never say "failed" — say "you now know exactly where the marks were lost." Emotional support before analysis in any reflection/postmortem flow. Buttons are verbs with direction ("Start closing the gap," not "Continue"). Always disclose ACCA independence near ACCA content.

CHECKLIST before shipping
[ ] system components only  [ ] tokens not hex  [ ] 4px grid  [ ] semantic icons  [ ] one hero number  [ ] motion + reduced-motion safe  [ ] honest empty states  [ ] 44px touch targets  [ ] evidence-tied, non-judgmental copy
